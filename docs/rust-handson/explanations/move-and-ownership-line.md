# 解説: 所有のラインは 1 本だけ

[02-ownership.md](../02-ownership.md) の Move（所有権の移動）の補足。

> 持ち主は 1 人、見学者は何人でも。所有のラインが 1 本だけだから、double free が起きない

## move とは「持ち主の札の付け替え」

```rust
let s1 = String::from("hello");
let s2 = s1;             // ← move。s1 はもう使えない
```

ヒープの `"hello"` 本体は動かない。動いたのは「これは私のもの」という札（所有権）。

```
move 前                              move 後
─────────                            ─────────
スタック         ヒープ              スタック         ヒープ
┌────────┐     ┌─────┐              ┌────────┐     ┌─────┐
│ s1 ────┼───→ │"hi" │              │ s1 (X) │     │"hi" │
│        │     └─────┘              │ s2 ────┼───→ └─────┘
└────────┘                          └────────┘
```

ある瞬間に「所有しているポインタ」は必ず 1 本。`s1` から `s2` へ「持ち主の札」が付け替わるイメージ。

## なぜ「1 本だけ」を強制するのか

答え: double free を防ぐため。

もし `s1` と `s2` の両方が同じヒープを所有していたら、スコープを抜けるときに両方が「自分が解放する」と思って `free` を 2 回呼んでしまう。これは C/C++ で頻発するクラッシュ要因。

Rust は「所有者は 1 人」を型システムで強制することで、コンパイル時に double free をゼロにする ―― ランタイムコストなしで。

## 借用が入ると矢印は増える

「ヒープを指すポインタ」自体は、借用 `&` を使えば複数あり得る。

```rust
let s = String::from("hi");
let r1 = &s;   // ← ヒープを指す参照
let r2 = &s;   // ← もう 1 本
```

```
スタック              ヒープ
┌────────┐          ┌─────┐
│ s  ────┼────────→ │"hi" │
│ r1 ────┼─────────→│     │
│ r2 ────┼────────→ │     │
└────────┘          └─────┘
```

ただしこれは「借りているだけ」で、`free` する責任を持っているのは `s` だけ。`r1`, `r2` は所有者ではないので、スコープを抜けても何も解放しない。

つまり Rust の所有権ルールを正確に言うと:

- 所有しているポインタ（解放責任を持つもの）は 1 本
- 借用しているポインタは複数 OK（ただし `&mut` なら 1 本だけ）

「持ち主は 1 人、見学者は何人でも」。

## 他言語との対比

| 言語 | 「持ち主」の概念 | double free をどう防ぐか |
|---|---|---|
| C / C++ | なし（生ポインタ） | 人間ががんばる。バグると死ぬ |
| C++ (`unique_ptr`) | あり（ライブラリ） | ライブラリのルールに従う限り防げる |
| Go / Java / Ruby / PHP / Python | なし（GC が回収） | そもそも自分で `free` しないので発生しない |
| Rust | あり（言語コア） | 型システムが「持ち主 1 人」を強制 |

GC 言語に慣れていると「double free? そんなの考えたこともない」となるが、それは GC が裏で「いつ誰のメモリを解放するか」をランタイムで判定してくれているから。Rust は GC を使わずに同じ安全性を得るために、「持ち主が誰か」をプログラマと型システムで明示している。

C++ の `std::unique_ptr` は「move 専用のスマートポインタ」で、Rust の所有権と似た発想。ただし C++ では生ポインタも `shared_ptr` も併用できるので、ルールから逸脱したコードも書けてしまう。Rust はそれを言語レベルで禁止している。

## `&mut` が 1 本だけ、も同じ哲学

後の節で出てくる借用ルール:

- 不変参照 `&T` は何本でも OK
- 可変参照 `&mut T` は 1 本だけ
- 両者を同時に持つことはできない

これも「同じ値に対する書き換え権限を持つ存在は 1 つだけ」というルールで、所有権の「持ち主は 1 人」と同じ哲学から来ている。所有 = 値の解放権、`&mut` = 値の書き換え権、と権限を分けて、それぞれに「1 つだけ」を強制する。

データ競合（複数の書き換え権が同時に存在する状態）を、所有権と同じ仕組みで弾いている。

## まとめ

- move は「ヒープのコピー」ではなく「持ち主の札の付け替え」
- 所有のラインは常に 1 本 → double free が起きない
- 借用は別ルートで複数 OK だが、所有者ではないので解放しない
- 「`&mut` も 1 本だけ」のルールも同じ哲学 ―― 書き換え権限を 1 つに絞ってデータ競合を防ぐ
- GC 言語が「持ち主の概念なし、ランタイムで回収」なら、Rust は「持ち主を型で追って、決定論的に解放」
