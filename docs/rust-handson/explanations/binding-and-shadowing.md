# 解説: 変数・値・束縛、mut とシャドーイング

[01-basics.md](../01-basics.md) の「シャドーイング」セクションの補足。

> 同じ束縛は同じ型でないといけない。シャドーイングは別物だから型を変えられる。

これが両者を見分ける一番分かりやすい違い。

## 「束縛 (binding)」と「変数 (variable)」の違い

他言語だと「変数」と言えば「名前と値の組み合わせ」をひとまとめにするが、Rust では分けて考える方が正確。

| 用語 | 意味 |
|---|---|
| 値 (value) | メモリ上の実体 (`5`、`"hello"`) |
| 変数 (variable) | 値を入れる場所（メモリ上の領域） |
| 束縛 (binding) | 名前と値を結びつけたもの |

`let x = 5` は「`x` という名前を、値 `5` に束縛する」操作。

## シャドーイングは「新しい束縛」、再代入は「同じ束縛の中身を変える」

```rust
let x = 5;            // 束縛①: x → 5
let x = x + 1;        // 束縛②: 新しい x → 6（束縛①は隠れる）
```

```rust
let mut x = 5;        // 束縛①: x → 5
x = x + 1;            // 束縛①の中身を 6 に変える（同じ束縛）
```

シャドーイングでは束縛①と束縛②は別物。たまたま同じ名前を使っているだけで、コンパイラから見ると完全に別の存在。

## 「別物」である決定的な証拠: 型を変えられる

```rust
let x = 5;            // i32
let x = "hello";      // &str（型を変えても OK）
let x = x.len();      // usize（さらに型を変える）
```

これがシャドーイングが「新しい束縛」である決定的な証拠。`mut` での再代入だと:

```rust
let mut x = 5;
x = "hello";          // ❌ 型不一致エラー
```

同じ束縛は同じ型でないといけない。シャドーイングは別物だから型を変えられる。

## メモリレベルでの違い

```rust
let x = 5;            // スタックに 5 を置き、x で参照（束縛①）
let x = x + 1;        // 別の場所に 6 を置き、新しい x で参照（束縛②）
                       // 束縛①の値 5 はまだスタックに居るが、もう参照する手段がない
```

`mut` の再代入:

```rust
let mut x = 5;        // スタックに 5 を置く
x = x + 1;            // 同じ場所の 5 を 6 に書き換える
```

メモリの動きが違う。シャドーイングは「別の場所に新しい値を作って、新しい束縛がそれを指す」、`mut` は「同じ場所を上書き」。

## なぜ言葉を区別するのか

Rust の設計思想:

- デフォルトは不変（束縛は `let` のみ。値を変えるなら `mut` を明示）
- 「変える」と「新しく作る」を区別したい
- シャドーイングは後者を表現するための機能

PHP / Ruby / Python のように「変数 = 値を入れ替えられる箱」という 1 つのモデルだと、この区別が見えなくなる。Rust では「束縛」という言葉を使うことで、シャドーイングが「新しい束縛を作っているだけで、再代入ではない」ことを言語化できる。

## 使い分け

| 操作 | やっていること | 型変更 | 用語 | 使う場面 |
|---|---|---|---|---|
| `let mut x = 5; x = 6;` | 同じ束縛の中身を更新 | 不可 | 再代入 | カウンタ、累積値、ループ変数 |
| `let x = 5; let x = 6;` | 新しい束縛を作る | 可能 | シャドーイング | 値を変換していく、型を変える |

例:

```rust
// mut: カウンタを増やす（同じ概念上の値を更新）
let mut count = 0;
for _ in 0..10 {
    count += 1;
}

// シャドーイング: 値を加工していく（型を変えながら変換）
let s = "  hello  ";
let s = s.trim();           // &str → &str（同じ型でも変換）
let s = s.to_uppercase();   // &str → String（型を変える）
let s = s.len();            // String → usize
```

意図が違う 2 つの操作を、構文レベルで区別している。

## 関数型言語からの影響

Rust の `let` は実は関数型言語（Haskell や OCaml）の `let` と同じ思想。これらの言語でも「束縛 (binding)」と言い、「再代入」という概念がそもそもない（不変が前提）。

```ocaml
let x = 5 in
let x = x + 1 in   (* 新しい束縛 *)
print_int x        (* 6 *)
```

Rust は「不変デフォルト + シャドーイング」を関数型から取り入れていて、これが PHP / Ruby / Python とは違う言語感の源になっている。

## スコープが切れた時の挙動

シャドーイングはスコープを抜けると元に戻る。

```rust
let x = 5;
{
    let x = x + 100;   // ブロック内でのシャドーイング
    println!("{x}");   // 105
}
println!("{x}");       // 5（外側の束縛が復活）
```

`mut` の場合はスコープに関係なく値が変わったまま:

```rust
let mut x = 5;
{
    x = 105;           // ブロック内で書き換え
}
println!("{x}");       // 105（変わったまま）
```

これも「シャドーイング = 新しい別物」「`mut` = 同じものの中身を変える」の違いが効いている例。

## まとめ

- Rust では「変数」「値」「束縛」を区別する
- シャドーイングは「同じ名前の新しい束縛」を作る（型も変えられる）
- `mut` は「同じ束縛の中身を変える」（型は固定）
- 「束縛」という言葉が Rust の不変性デフォルトの思想を表現している
- 使い分け: 値を更新するなら `mut`、値を変換していくならシャドーイング
