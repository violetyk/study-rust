# 解説: 参照と参照外し

[02-ownership.md](../02-ownership.md) の借用セクション（参照外し `*`）の補足。

> `&` で借りて `*` で外す。でも実際は `*` を書かなくていい場面の方が多い ―― Deref トレイトと自動 deref のおかげ

## 参照とは何か

`&T` は「`T` 型の値が置かれているアドレス」を保持する型。スタック上の小さなデータ（ポインタ）として実体を持つ。

```rust
let x: i32 = 5;
let r: &i32 = &x;
```

メモリ上は:

```
スタック
┌─────────────┐
│ x: i32 = 5  │ ← アドレス 0x...A0
│ r: &i32 ────┼──→ 0x...A0
└─────────────┘
```

`r` の中身は「アドレス 0x...A0」というビット列。`*r` を書くと「そのアドレスに行って中身を読む」操作になる。

## `*` の基本動作

### 値を取り出す

```rust
let x = 5;
let r = &x;
let v = *r;      // v は i32 = 5（Copy なのでコピーされる）
println!("{v}");
```

注意: `*r` で「中身を取り出す」のは Copy 可能な型なら問題ないが、Copy でない型（例: `String`）に `*r` を書くと所有権を奪う動作になり、コンパイルエラーが出る。

```rust
let s = String::from("hi");
let r = &s;
let owned = *r;  // ❌ cannot move out of `*r` which is behind a shared reference
```

ヒープを持つ型を `*` で取り出したいなら `.clone()` する。

### 書き換える

```rust
let mut y = 10;
let mr = &mut y;
*mr += 1;
println!("{y}");   // 11
```

`mr = 11` だと「`mr` 自体（参照そのもの）を別の場所を指すように変える」という意味になってしまう。書き換えたいのは「参照の先」なので `*mr = ...` と書く。

## なぜ `*` を書かなくていい場面が多いのか

Rust には「自動 deref（auto-deref / Deref coercion）」という仕組みがある。

### メソッド呼び出し時の自動 deref

```rust
let s = String::from("hello");
let r = &s;
r.len();           // ← (*r).len() の糖衣構文
```

メソッド呼び出し (`.len()` など) では、コンパイラが必要なだけ自動で `*` を挿入してくれる。`&String` → `String` → `&str` のように、辿れるところまで辿って一致するメソッドを探す。

### `println!` などのフォーマッタ

```rust
let x = 5;
let r = &x;
println!("{r}");   // ✅ 5 と出る
println!("{}", *r); // ✅ こちらでも 5。どっちでもいい
```

`Display` / `Debug` トレイトは `&self` を受け取る実装になっているので、参照のままでも値のままでも動く。

### 関数引数の Deref coercion

```rust
fn takes_str(s: &str) { println!("{s}"); }

let s = String::from("hi");
takes_str(&s);     // ← &String が &str に自動変換される（Deref coercion）
```

`String` は `Deref<Target = str>` を実装しているので、`&String` が `&str` を期待する場所に渡せる。

## `Deref` トレイトとは

`Deref` は「`*` を書いたときに何を返すか」を定義するトレイト。標準ライブラリに以下のような型がたくさん実装している:

| 型 | `Deref::Target` | `*` で何が出るか |
|---|---|---|
| `&T` | `T` | `T` の値 |
| `Box<T>` | `T` | `T` の値（ヒープから） |
| `String` | `str` | `str`（厳密には `&String` → `&str` の coercion） |
| `Vec<T>` | `[T]` | `[T]` スライス |
| `Rc<T>` | `T` | `T` |
| `Arc<T>` | `T` | `T` |

これがあるから、`Box<T>` も `Rc<T>` も「ポインタっぽい型」として `*box` のように書ける。

```rust
let b: Box<i32> = Box::new(42);
let n: i32 = *b;     // b の中身を取り出す（Copy なので可能）
println!("{n}");     // 42
```

## `*` を明示的に書く必要がある場面

自動 deref は便利だが、以下の場面では自分で `*` を書く必要がある:

1. 参照経由で値を書き換える: `*mr = 10;` `*mr += 1;`
2. Copy 型を値として取り出す: `let v: i32 = *r;`
3. パターンマッチで参照を剥がす:
   ```rust
   let v = vec![1, 2, 3];
   for &x in &v {        // &x で参照を剥がして x: i32 にする
       println!("{x}");
   }
   ```
4. 比較のとき、片方が参照でもう片方が値:
   ```rust
   let x = 5;
   let r = &x;
   if *r == 5 { /* ... */ }   // r == 5 はエラー
   ```

## 他言語との対比

| 言語 | 参照の取り方 | 参照外し | 自動 deref |
|---|---|---|---|
| C | `&x` (アドレス取得) | `*p` | なし（書かないと NULL でクラッシュ） |
| C++ | `&x` / リファレンス `T&` | `*p`（リファレンスは不要） | リファレンス型は自動 |
| Go | `&x` | `*p` | フィールドアクセスのみ自動（`p.field` は `(*p).field`） |
| Java / Ruby / Python / PHP | (常に参照、明示的な `&`/`*` なし) | (不要) | 参照しか扱わないので無関係 |
| Rust | `&x` / `&mut x` | `*r` | メソッド・フォーマッタ・関数引数で自動 |

Rust は C/Go と同じ `&` / `*` シンタックスを採用しつつ、ボロチェッカーで安全性を保証し、自動 deref で書き味を改善している。「明示的だけど省略可」という設計。

## まとめ

- `&T` はアドレスを持つ小さな値。`*r` で「そのアドレスの先」を取り出す
- 書き換えるには `*mr = ...` のように明示的に参照外しが必要
- Rust は「自動 deref」のおかげで `*` を書く場面が少ない（メソッド呼び出し・フォーマッタ・関数引数で自動挿入）
- `Deref` トレイトを実装した型（`Box`, `Rc`, `String`, `Vec` など）は `*` で中身にアクセスできる
- 明示的に必要なのは「書き換え」「Copy 型の値取り出し」「パターンマッチで剥がす」「比較」くらい
